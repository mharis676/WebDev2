                <h1>Summary of Lecture 21</h1> 
                <ul>
                    <li> Cascade: Many selectors are used to style a single element and CSS solves this conflict by Cascade.</li>
                    <li>Cascade Algorithm : It determines which property of CSS will be applied to the element. The cascade algorithm has 4 distinct stages.</li>
                    <li> Position and order of appearance: If the same selector, then the last one will be used</li>
                    <li> Origin: authored CSS has priority</li>
                    <li> Important: When !important is used, it has the highest priority.</li> 
                    <li> Specificity: Selector with high specificity is applied to the HTML element.</li>
                    <li> Order of Specificity: Inline Style > ID selector > Class or Attribute selector > Element Selector > Universal Selector</li>
                    <li> Calculating Specificity: Value of Each Selector: Universal is 0, 1 for each element selector or pseudo-element, 10 for each class value (or pseudo-class or attribute selector), 100 for each ID value. Inline Style has 1000 specificity. Just add value to find specificity</li>
                </ul>
